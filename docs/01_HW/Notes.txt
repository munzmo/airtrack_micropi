##Connect Problem and Solution
###Problem
1. active ModemManager
    The Linux service ModemManager accesses all newly recognized USB serial devices by default to check for possible modems. This caused the ESP32 USB UART to be occupied for a short time.
2. Authorizations
    The user was not a member of the dialout group. This meant that read/write access to /dev/ttyUSB0 was not possible, even though the device was recognized correctly.

###Solution
    - ModemManager permanently disabled (mask)
    - User explicitly added to the dialout group
    - New session started with active group membership
    - After that, stable access to /dev/ttyUSB0 and successful MicroPython REPL
